# kraken-cursor: setup-copilot

**Announce at start:** "I'm using kraken-cursor-setup-copilot to generate Copilot instructions."

Reads skills from the SkillsOfTheKraken repo (`skills/` and `kraken-cursor/`), distills each into Copilot-friendly rules, and writes `.github/copilot-instructions.md` to `{repoRoot}`. Once committed, Visual Studio and VS Code Copilot enforce the same standards automatically.

This skill is **VS Copilot focused** — it does not configure Cursor agent skills (those install separately under `~/.cursor/skills/`).

## Pre-flight

1. Resolve `{repoRoot}` = current client repo root (where `.sln`, `_package-request.json`, or `client.json` lives).
2. Confirm `gh` CLI is authenticated (`gh auth status`).

## Steps

### Step 1 — Fetch all skills from the repo

```powershell
$repo     = "SeayMonster/SkillsOfTheKraken"
$folders  = @("skills", "kraken-cursor")
$contents = @{}

foreach ($folder in $folders) {
    $entries = gh api "repos/$repo/contents/$folder" --jq '.[] | select(.type=="dir") | .name'
    foreach ($skill in $entries) {
        $key = "$folder/$skill"
        $raw = gh api "repos/$repo/contents/$folder/$skill/SKILL.md" --jq '.content' |
               ForEach-Object { [System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String($_)) }
        if ($raw) { $contents[$key] = $raw }
    }
}
```

### Step 2 — Distill each skill into Copilot instructions

For each skill, read its content and extract rules that Copilot can apply when writing or reviewing code.

**Include skills that define:**

- Coding style, patterns, naming conventions
- Architecture rules, class/method structure
- Database access patterns
- Logging conventions
- Technology-specific patterns (C#, SQL, VB, etc.)

**Skip skills that are agent workflow only** (no coding rules):

- QA orchestration (`qa-run`, `qa-uat`, `qa-init`, `qa-*`)
- Deployment packaging, client onboarding, Notion setup
- Pin project, desktop-vscode-bridge, setup-copilot itself
- Skill creation or Cursor/Claude configuration
- Skills that only describe when to invoke a tool

### Step 3 — Write the file

Create `.github/copilot-instructions.md` under `{repoRoot}`:

```powershell
$outDir  = Join-Path (Get-Location).Path ".github"
$outFile = Join-Path $outDir "copilot-instructions.md"
if (-not (Test-Path $outDir)) { New-Item -ItemType Directory -Path $outDir | Out-Null }
```

Format the file as:

```markdown
# Copilot Instructions
<!-- Generated by kraken-cursor-setup-copilot. Re-run to update. -->

## [Skill Name]
[Direct rules distilled from the skill — imperative, specific, actionable]
- Rule 1
- Rule 2

## [Next Skill Name]
...
```

Write rules as **direct imperatives** ("Use X", "Always Y", "Never Z") — not as descriptions of what the skill does.

### Step 4 — Confirm

Tell the user:

> Written to `.github/copilot-instructions.md`. Commit it and Copilot will follow these rules in Visual Studio and VS Code automatically.

## Notes

- Safe to re-run — overwrites the file with the latest skills from the repo
- The file should be committed to the repo so teammates get it on pull
- Only coding-relevant rules are included — agent workflow skills are skipped
- Cursor users still use `kraken-cursor-*` skills for agent workflows; this file is for Copilot in VS/VS Code
